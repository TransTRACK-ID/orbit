# Product

## Register

product

## Users

Project managers who oversee software development teams. They use Orbit to track tasks, coordinate with AI agents, manage workspaces, and monitor project progress. They need clarity at a glance, efficient task organization, and confidence that nothing falls through the cracks. Primary context: desktop, multiple projects, frequent context-switching between boards and settings.

## Product Purpose

Orbit is a project management platform that blends traditional kanban task tracking with AI agent orchestration. It exists to help teams move from "task lists" to "agent-assisted workflows" where tasks can be assigned to AI agents that process them automatically. Success looks like project managers spending less time on status updates and more time on strategic decisions.

## Brand Personality

- **Assured**: Confident without being arrogant. The UI makes decisions feel easy.
- **Organized**: Everything has its place. No visual noise, no ambiguity.
- **Warm-tech**: Technical capability wrapped in approachable, human-friendly design.

## Anti-references

- **Developer-focus approach**: Avoid terminal-like aesthetics, raw monospace UI, unpolished data dumps, or "just expose the database" interfaces. This is a PM tool, not a dev tool.
- **Jira complexity**: Avoid labyrinthine navigation, endless configuration screens, and feature bloat that obscures the work.
- **Trello oversimplicity**: Avoid feeling like a toy. The agent integration and workspace hierarchy need to feel substantial.
- **Linear's icy minimalism**: Avoid sterile, cold interfaces that feel disconnected from the humans using them.

## Design Principles

1. **Clarity over density**: Every screen should answer "what needs my attention?" in under 3 seconds.
2. **Progress is visible**: Status changes, agent activity, and task movement should be immediately apparent without hunting.
3. **Warm precision**: Sharp enough for power users, warm enough for daily use. No cold corporate sterility.
4. **Agent presence is ambient**: AI agent indicators should feel like helpful colleagues, not intrusive robots.
5. **Frictionless flow**: The most common actions (create, move, assign) should require the fewest interactions.

## Accessibility & Inclusion

WCAG 2.1 AA baseline. Support for keyboard navigation and reduced motion preferences. Color is not the sole indicator of status or priority.
